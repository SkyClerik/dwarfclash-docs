# Дополнительные команды Doxygen

```
  Doxygen использует специальные команды, которые начинаются с \ или @, внутри твоих XML-комментариев (/// или /** ... */).

  1. \todo (или @todo)

   * Назначение: Используется, чтобы пометить код или часть документации, которая требует дальнейшей работы, улучшения или исправления.
   * Что делает Doxygen: Он собирает все \todo элементы из всего проекта и создаёт отдельную страницу "Todo List" в сгенерированной документации.
     Это позволяет легко отслеживать незавершённые задачи.
   * Синтаксис:

   1     /// <summary>
   2     /// Этот метод делает что-то важное.
   3     /// </summary>
   4     /// <param name="param1">Описание параметра.</param>
   5     /// <returns>Возвращает результат.</returns>
   6     /// \todo Добавить обработку граничных случаев для param1.
   7     /// \todo Оптимизировать этот цикл, он работает медленно.
   8     public void MyMethod(int param1) { /* ... */ }
```

\todo Добавить обработку граничных случаев для param1.

```
  2. \note (или @note)

   * Назначение: Используется для добавления важной заметки или пояснения, которое не является частью основного описания метода или класса. Обычно
     Doxygen оформляет такие заметки как отдельный блок.
   * Синтаксис:

   1     /// <summary>
   2     /// Этот метод выполняет сложную операцию.
   3     /// </summary>
   4     /// \note Убедитесь, что перед вызовом этого метода все зависимости инициализированы.
   5     /// \note Для лучшей производительности рассмотрите асинхронный вызов.
   6     public void ComplexOperation() { /* ... */ }
```

\note Убедитесь, что перед вызовом этого метода все зависимости инициализированы.
\note Для лучшей производительности рассмотрите асинхронный вызов.

```
  3. \warning (или @warning)

   * Назначение: Очень похоже на \note, но используется специально для предупреждений о потенциальных проблемах, подводных камнях или критических
     моментах, которые должен знать пользователь кода. Часто Doxygen отображает их с особым значком или цветом.
   * Синтаксис:

   1     /// <summary>
   2     /// Удаляет все данные пользователя.
   3     /// </summary>
   4     /// \warning Эта операция необратима и приведет к потере всех пользовательских данных. Используйте с крайней осторожностью.
   5     public void DeleteAllUserData() { /* ... */ }
```

\warning Эта операция необратима и приведет к потере всех пользовательских данных. Используйте с крайней осторожностью.

```
  4. \remarks (или @remarks)

   * Назначение: Для более обширных замечаний, дополнительной информации или пояснений, которые не умещаются в summary. В XML-комментариях это
     эквивалент <remarks>...</remarks>.
   * Синтаксис:

   1     /// <summary>
   2     /// Этот метод вычисляет ...
   3     /// </summary>
   4     /// \remarks
   5     /// Детальное описание алгоритма и его производительности.
   6     /// Объяснение, почему был выбран именно этот подход.
   7     /// </remarks>
   8     public float CalculateValue() { /* ... */ }
```
\remarks
Детальное описание алгоритма и его производительности.
Объяснение, почему был выбран именно этот подход.
</remarks>

```

  ---

  Как это работает:

  Когда ты запускаешь Doxygen, он сканирует все твои комментарии.
   * Для \todo, \bug, \deprecated, \test он создаёт отдельные страницы со списками. (Мы уже включили GENERATE_TODOLIST = YES в твоём Doxyfile).
   * \note, \warning, \remarks отображаются непосредственно в документации к тому элементу, к которому они относятся.
   ```
